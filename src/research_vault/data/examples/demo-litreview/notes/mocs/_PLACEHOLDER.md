# mocs/ — Maps of Content

Each file is a MOC (Map of Content) linking related notes into a navigable index.
Orphan notes (not linked in any MOC) are flagged by the synthesis critic but not blocked.

**Naming convention:** `<topic-slug>.md`

**Required frontmatter:**
```yaml
---
type: mocs
title: <MOC title>
description: <one-sentence summary of the note (optional; WARN if empty)>
---
```
